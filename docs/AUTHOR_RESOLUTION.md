# 作者归属判定

RAMSAY 使用同一条解析链为卡片详情、作者排行榜、系列投票作者验证和召唤词作者验证提供作者信息，避免各功能各自截取卡号前三位。

## 判定优先级

1. 管理员文本规则
2. 管理员卡号区间
3. CDB 署名
4. `strings.conf` 保守兜底
5. 未知作者

每张 DIY 卡同时带有 `author_source`、`author_source_label`、`author_rule_id`、`author_matched_on` 和 `author_matched_value`。卡片详情会显示判定来源，便于追查误判。

## 管理员卡号区间

作者码不能直接对 SQLite 整数卡号执行普通的 `startsWith`。前导零和卡号总位数都是区间的一部分：

- 作者码 `10`、总位数 `8` 可匹配显示为 `01000360` 的卡号。
- 作者码 `130`、总位数 `8` 匹配 `13000745`。
- 作者码 `1300`、总位数 `9` 匹配 `130001000`，不会抢走上一条八位卡号。
- 作者码 `2025`、总位数 `8` 匹配 `20250329`。

后台可选择以下任一种方式：

- 填写作者码和卡号总位数；
- 填写包含端点的显式起止卡号，适合不规则区间。

每条映射使用稳定 ID 管理；同一个作者码可以保存多个显式区间，并分别设置作者和优先级。升级时会自动移除旧数据库对 `card_prefix` 的唯一约束，同时保留原记录 ID 和人工字段。

旧记录没有卡号总位数时，一至三位作者码按八位卡号处理；四位作者码会根据 CDB 中八位、九位候选区间的实际卡片数量推断；五至十六位前缀使用“前缀长度 + 5”（最多十六位）的保守默认值。管理员保存后仍应填写明确值，避免今后数据分布改变推断结果。

## 管理员文本规则

文本规则用于没有稳定卡号区间的散卡或跨区间系列。规则包含：

- CDB 来源文件，可留空表示全部；
- 字段：`name`、`desc`、`str1` 至 `str16`，或谨慎使用 `any`；
- 匹配方式：包含、全文相等、整行相等；
- 匹配值、目标类型（作者归属或人工系列分组）、目标名称、优先级、大小写选项、启用状态和备注。

例如，为 `no42.cdb` 中描述里具有独立一行 `Copyright 本体` 的卡指定作者，可设置：

```text
来源：no42.cdb
字段：desc
方式：整行相等
值：Copyright 本体
目标类型：作者归属
目标名称：管理员确认的规范作者名
```

规则按优先级从高到低处理；相同优先级优先使用更严格、匹配值更长的规则。文本规则是人工例外，优先于卡号区间。

作者目标参与作者详情、作者排行榜、投票发起人和召唤词作者身份验证。系列目标存入独立的 `manual_series_*` 元数据，用于卡片详情和首页同系列推荐；它绝不会进入作者榜或作者身份白名单。

人工系列分组不会改写 CDB 的数值 `setcode`，因此也不会悄悄改变现有的系列投票和禁卡统计语义。需要修改游戏内 setcode 时，应直接治理 CDB 数据，而不是用显示分组冒充数值 setcode。

## CDB 与 strings.conf

- 解析器统一读取 `desc` 与 `str1` 至 `str16`，并先把 CRLF、CR、LF 统一成 LF。
- CDB 署名按行解析，识别常见的 `DoItYourself`、`DoltYourself`、`DoItByYourself`、`DIY`/`DlY` 变体，以及保守的 `作者--DoItYourself`、`「DoItYourself」by 作者`、独立行 `CardDesign by 作者`；图片、PSCT、脚本等其他贡献者不会自动覆盖主作者。
- 管理员维护的作者名只移除空白和控制字符，不做英文空格、括号等启发式截断。
- CDB 署名若命中管理员维护的别名，会归一到规范作者名。
- `strings.conf` 的十进制作者码和十六进制 setcode 是不同命名空间。自动识别会排除 hex、QQ 号和冲突作者码，只新增无冲突候选，绝不覆盖人工记录。

## 重复卡号与缓存

`CARD_DATABASE_PRIORITY` 明确同一 ID 出现在多个 CDB 时的权威来源。默认顺序是 `no81.cdb`、`no42.cdb`；未列出的文件按文件名稳定排序。可在 `config.user.php` 覆盖：

```php
define('CARD_DATABASE_PRIORITY', json_encode(['no81.cdb', 'no42.cdb']));
```

作者榜缓存指纹包含：

- CDB、`strings.conf`、`lflist.conf` 的文件名、大小和修改时间；
- 全部作者区间和文本规则；
- 简略模式、排除数据库和数据库优先级配置。

作者区间或文本规则发生增删改、启停时，文件缓存和当前请求的内存规则缓存都会立即失效。

## 首次部署

部署新版本前仍应按常规流程备份 `data/ramsay.db`。首次请求会自动完成两项兼容迁移：移除旧的单前缀唯一约束，以及为旧文本规则补充目标类型/目标值；原映射 ID、作者、别名、联系方式、备注和时间戳都会保留。迁移使用 SQLite 写锁和二次结构检查，允许多个部署后请求安全等待首个迁移请求。

PHP 进程需要继续拥有 `data/` 与 `data/cache/` 的写权限。作者榜缓存现在使用单文件封套与锁文件 `author_hall_of_fame.lock`，旧的分离元数据缓存会在下次失效时清理。
